# table展示居中法

特点

  1. 通常是能胜任一些文字居中，但display: table的局限性非常大。
  2. 它的外层结构不能随意设置定位流，因为会导致display失效。
  3. 内层结构容易出现宽、高、边距无效问题。
  4. 往往需要嵌套多层标签去解决上面的问题，这导致代码会很难维护。
  5. 除了可以兼容到ie8外，我不推荐平时用这种方法去居中。
.wrap {
  display: table;
  .box {
    display: table-cell;
    vertical-align: middle;
  }
}

# flex展示居中法

特点：

  1. flex具有自动收缩功能，可以胜任响应式布局。
  2. 传统的居中方法都需要在子元素上设置属性，而flex提出了一种抽象模式，通过父元素属性定位子元素。
  3. 我认为这个模式是优秀的，在后来的grid 和 webkit-box上都提现了这个模式。
  4. 缺点是ie9及以下不兼容，
  5. felx 把子元素都升级为弹性盒子，覆盖了 block 和 inline-block原有的特性。
.wrap{
  display: flex;
  justify-content:center;
  align-items: center;
}

# grid展示居中法

特点：

  1. 比flex更加抽象的居中方式
  2. 列表居中推荐使用
  3. 对ie、Opera、还有一些手机系统的webview支持不太好
.wrap{
  display: grid;
  justify-content:center;
  align-items: center;
}
